## 📌 Overview

Black-box analysis of a synthetic employee dataset generated by a classmate.  
The dataset was treated as "alien data" — no prior knowledge of how it was generated.  
The goal was to parse, clean, reverse-engineer its distributions, and compare findings against the actual generation logic.

## 🔬 Analysis Performed

### 1. Exploratory Data Analysis (EDA)
- Schema inspection (`dtypes`, `info()`, `describe()`)
- Null value detection — **0 missing values found**
- Uniqueness ratio per column to identify keys and constant fields
- Chunked loading strategy for the 1M-row dataset

### 2. Distribution Analysis
Skewness/kurtosis analysis applied to all numerical columns:

| Column | Inferred Distribution |
|---|---|
| `days_service` | Uniform — `randint(1, 1827)` |
| `zip_code` | Uniform — `randint()` |
| `base_salary` | Log-normal / Beta `[30k, 200k]` |
| `bonus_percentage` | Normal — `N(5, 2)` clipped at 0 |
| `performance_score` | Normal — `N(75, 10)` clipped at 100 |
| `vacation_days` | Discrete Normal — `N(15, 4)` |
| `sick_days` | Poisson — `λ ≈ 5` |

### 3. Categorical Analysis
- Frequency distributions for `department`, `job_title`, `state`, `gender`, `education`, `employee_level`, `shift`, `work_location`, `status`
- Detection of weighted vs flat `random.choice()` usage
- `country` identified as a hardcoded constant (`"USA"`) with no analytical value

### 4. Clustering
- KMeans clustering on scaled numerical features
- Elbow method to select optimal k
- PCA projection for 2D cluster visualization

### 5. Correlation Analysis
- Heatmaps for both small and big (sampled) datasets
- **Finding:** Near-zero correlation across all numerical variables — confirms fully independent generation

---

## 🧠 Key Findings

- The dataset is **completely synthetic**, generated using Faker + NumPy distributions
- Variables were generated **independently** — no real-world correlations exist
- `email` is not a reliable primary key (collision ratio ~70%)
- `zip_code` should be treated as categorical, not numeric
- Suitable for **pipeline testing and benchmarking**, not for predictive modeling

Full reverse-engineering map in [`report/findings.md`](report/findings.md).
